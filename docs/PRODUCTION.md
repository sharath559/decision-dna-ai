## Production Migration Path

### Phase 1: API Service Layer (1-2 weeks)

**Goal:** Decouple the analysis engine from the Streamlit UI for headless access.

```python
# FastAPI gateway (future)
from fastapi import FastAPI
from src.services.mutation_engine import run_full_analysis

app = FastAPI(title="DecisionDNA AI API")

@app.post("/api/v1/analyze/{case_id}")
async def analyze_case(case_id: str):
    result = run_full_analysis(case_id)
    return result["mutation_report"].model_dump()
```

- Extract core logic from `app.py` into API endpoints
- Add JWT/OAuth authentication middleware
- Add OpenAPI documentation (auto-generated by FastAPI)

### Phase 2: Persistent Storage (1 week)

**Goal:** Replace in-memory audit store with durable database.

- Connect `AuditMCP` to Cloud SQL (PostgreSQL)
- Add SQLAlchemy models mirroring Pydantic schemas
- Enable audit trail queries across restarts

### Phase 3: Enterprise Gemini Integration (1 week)

**Goal:** Move from AI Studio to Vertex AI for SLA-backed LLM access.

- Set `GOOGLE_GENAI_USE_VERTEXAI=true`
- Configure service account with Vertex AI User role
- Add request retry with exponential backoff
- Add response caching for repeated case analyses

### Phase 4: Observability (1 week)

**Goal:** Production monitoring and alerting.

- Cloud Trace for latency tracking across agent pipeline
- Cloud Monitoring dashboards for mutation score distribution
- Alerting on security scan failures (potential attacks)
- BigQuery Agent Analytics plugin for ADK agents

### Phase 5: Enterprise Security (2 weeks)

**Goal:** Full compliance posture.

- Identity-Aware Proxy (IAP) for access control
- VPC Service Controls for data perimeter
- Cloud KMS for audit trail signing
- RBAC with predefined roles (auditor, admin, viewer)

## What's Already Production-Grade

The following components require **zero changes** for production deployment:

1. **Pydantic v2 Schema Layer:** 14 models with strict validation, used at every agent boundary
2. **SecurityAgent:** 21 detection patterns, PII redaction, and Gemini-enhanced classification
3. **Mutation Scoring Engine:** Deterministic, reproducible, mathematically verified
4. **Docker + Cloud Run Pipeline:** CI/CD with GitHub Actions, automated testing
5. **Hybrid LLM Architecture:** Graceful degradation when Gemini is unavailable
6. **Cryptographic Audit Hashing:** SHA-256 genome hashes for tamper evidence
